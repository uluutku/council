# Mobile Backend Contract

Council mobile uses the same Supabase project as the web app. The mobile client uses the public
anon key plus the authenticated user session; it does not use service-role credentials.

## Storage

| Bucket                | Access                                     | Mobile use                        |
| --------------------- | ------------------------------------------ | --------------------------------- |
| `profile-avatars`     | private, profile visibility RLS            | profile avatar upload/read        |
| `persona-avatars`     | private, owner RLS                         | custom persona avatar             |
| `message-attachments` | private, member RLS and staged upload RPCs | human chat image/file attachments |
| `ai-chat-images`      | private, owner RLS and staged upload RPCs  | AI image understanding            |
| `ai-chat-documents`   | private, owner RLS and staged upload RPCs  | AI PDF/TXT/Markdown understanding |

Signed URLs are treated as memory-only data and are not persisted.

## Realtime

| Topic                                      | Payload                                         | Authorization      | Mobile use                              |
| ------------------------------------------ | ----------------------------------------------- | ------------------ | --------------------------------------- |
| `user:{user_id}:inbox`                     | identifier-only conversation events             | exact owner topic  | inbox refresh, foreground notifications |
| `conversation:{conversation_id}`           | identifier-only message/reaction/receipt events | persistent members | conversation reconciliation             |
| `conversation:{conversation_id}:ephemeral` | typing start/stop only                          | members            | typing indicators                       |

Realtime is a hint. The mobile app reconciles through RPC reads after events, reconnects, resume,
and sequence gaps.

## AI Edge Function

`ai-chat` is called with `Authorization: Bearer <user access token>` and `apikey: <anon key>`.
The request contract is the existing AI generation JSON: `conversation_id`, `client_message_id`,
`content`, optional `attachment_ids`, optional `document_attachment_ids`, and optional
`context_import`. The response is strict SSE with `start`, `delta`, and exactly one terminal
`done`, `proposal_done`, or `error` event.
